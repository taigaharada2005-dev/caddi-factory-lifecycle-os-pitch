# CaddiPlant ／ 爆速工場（Factory Launch OS）

> 製品仕様・生産量・立地条件を入れると、工場レイアウト・設備構成・調達計画・施工計画・試運転計画・量産立ち上げKPIまで一気通貫で生成し、実行パートナーまでつなぐ「工場立ち上げOS」。

CADDiインターン課題「10年後に単年度売上2000億円規模になる、製造業の課題を解決するプロジェクト」（C班）から生まれたアイデアを、インターン終了後も個人プロジェクトとして継続開発しているリポジトリです。

## プロトタイプの機能

左で要件を入力すると、右側が**フェーズ・ステッパー**（提案・探索 → 設計図面 → 3Dシミュレーション → 立地・調達 → 事業性評価）で1フェーズずつ表示される。要件を変えると全フェーズのデータが連動再生成される。「詳細な図面ができてから立体をつくる」流れに合わせ、**設計図面フェーズを開くまで3Dシミュレーションはロック**される。

- **AIヒアリング** — 自然文で要望を話すと要件が自動反映（現状はデモ用モック。実装時はサーバ経由で Claude API `claude-opus-4-8` が自然文→構造化要件を抽出）
- **要件入力** — 製品／生産量／公差／新設のきっかけ／立地／稼働形態／必要認証／建設予算
- **2Dレイアウト** — 工程ブロックをドラッグで並べ替え、動線効率をリアルタイム評価（逆流検知）
- **設計探索エンジン（Design Search）** — GA風に複数案を生成→評価→改善する探索ログ＋収束カーブ＋探索空間＋推奨案。「AIが答える」のではなく「探索している」ように見せる（工場立ち上げを"探索できる設計問題"へ）
- **設計図面（配置図）** — 敷地境界・トラックヤード・事務所・駐車場・寸法線・表題欄・凡例つきのSVG自動生成。搬送ラインは単純な1直線ではなく、**分岐（加工A/B）→合流→検査→リワーク**を直交ルーティングで描画し、設備シンボル・構造柱・扉スイング・安全通路・バッファを備える
- **3D工場シミュレーター（Three.js）** — threejstips 準拠。エリア（原材料/加工/検査/包装/出荷）・設備（コンベア/ロボットアーム/検査ゲート/タンク/棚/AGV/作業者）・流れ（分岐・合流・リワークを通る箱アニメ）の3層。ボトルネック設備が赤く明滅。canvas上にHTML/CSSの **AI Optimization Panel**（Layout Score / Lead Time / Throughput / Bottleneck Risk / Worker Distance）＋ [Generate] [Simulate] [Optimize] [Reset View]。Optimize で動線が整理され数値が改善する Before/After
- **案比較・意思決定** — 推奨案／低コスト案／高速立上げ案を横並び比較（月産能力・建設投資・延床・投資回収・立上げ期間・人員・適合度、行ごとにベスト値をハイライト）。**目標未達アラート**（生産能力／予算／認証／動線を赤・黄・緑）、**数字の根拠（前提）開示**、**感度分析スライダー**（製品単価・生産量・人件費を±30%動かすと投資回収が即再計算）
- **SCM最適立地マップ** — 地域ごとの候補地を5指標（サプライヤー近接・物流港湾・人材・コスト・災害地政学）で採点し地図上に可視化、最適地をハイライト
- **業者マッチング** — 建設・EPCパートナーと設備メーカー候補を適合度・概算・納期つきで提示
- **インパクト評価** — この工場を作った場合の影響試算：年間生産能力／売上貢献／投資回収年数（累積キャッシュフローカーブ、黒字化点マーカー、ホバー付き）／直接雇用／地域経済波及／CO₂排出。「稼働1ヶ月前倒し＝約X億円のCF前倒し」という**爆速の価値の定量化**つき

### 開発体制（エージェント・オーケストレーション）
Claude Code を親エージェントとし、**Codex**（実装）・**agy**（"間違っている前提"での敵対的コードレビュー）を道具として使う。例：設計探索エンジンと3DシミュレーターはCodexが実装→agyが敵対レビュー→Claudeが実測（探索エンジンは576シナリオ）・修正・統合。定期的にCodex/agyへ全体俯瞰レビューをさせて改善方針を更新する。

## 方向性（インターン後）

詳細は [docs/strategy.md](docs/strategy.md) を参照。

- ポジション：EPC／デジタルツイン／モジュール建設／CQV が分断されている間隙を突く「**AI設計 → デジタルツイン検証 → 実行ネットワーク**」の一気通貫
- MVP：**工場立ち上げ診断＋レイアウト生成＋シミュレーション**（いきなり「建てます」はやらない）
- 初期ターゲット：**食品工場**（工程がパターン化しやすく、衛生区画・動線・人員の最適化余地が大きい。日立プラントサービスのAIレイアウト事例あり）
- 収益：初期診断／レイアウト・シミュレーション／実行PM支援（総投資額の1〜3%）／紹介手数料／SaaS／成果報酬
- モート：業界別標準工程・設備能力/価格/納期・施工実績・立ち上げトラブル等の**工場データ複利**
